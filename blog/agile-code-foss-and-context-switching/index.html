Agile code, FOSS and context switching

 Wed Nov 12 2025 00:00:00 GMT+0000 (Greenwich Mean Time)

Developing 11tyCMS has taught me so much about the impact of agile code and productivity. In early alpha, the code base was messy, inconsistent and difficult to follow. There were many flaws and excentricisities that made it extremely difficult to work with: all causing frequent bugs, slowing down debugging and draining energy.

I had spent so much time in "MVP mode" that I lost sight of the quality I was aiming for. It was fair enough, with my work commitments, the energy and time I had to dedicate to this project meant it was difficult. But knowing I would open the source code up motivated me to ensure that any contributors had a code base to work with that was lean, understandable and fun to work with.
